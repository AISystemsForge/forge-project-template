# Repository Naming

A repository name should describe its subject, system, or project. It should not
primarily advertise the platform, vendor, or institution through which the work
was learned or delivered.

Use lowercase kebab-case. Choose a name that is concise, stable, descriptive,
and free of unnecessary organization prefixes.

Preferred names include:

- `claude-code-software-engineering`
- `machine-learning-specialization`
- `generative-ai-engineering`
- `rag-evaluation-platform`
- `llm-observability`

Avoid names such as:

- `coursera-claude-code`
- `udemy-python`
- `edx-machine-learning`

The provider, institution, instructor, certificate, and course URL belong in
the repository documentation when they are relevant. They do not need to be
encoded in the repository name.

Prefer terminology that is likely to remain accurate as the project evolves.
Do not include a technology merely because it was used in the first
implementation when the repository represents a broader system or subject.
