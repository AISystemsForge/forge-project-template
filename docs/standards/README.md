# Engineering Standards

This directory contains the default repository and contribution conventions for
AISystemsForge. They are intentionally small, technology-neutral, and applicable
to educational, research, open-source, and production-oriented work.

These standards define shared defaults, not an approval process. A repository
may document a different choice when its collaborators, risk, or delivery model
requires one.

## Standards

- [Engineering governance](governance.md) defines decision authority, conflict
  resolution, exceptions, and the documented-first workflow.
- [Repository naming](repository-naming.md) describes how repositories receive
  concise and stable names.
- [Branch strategy](branch-strategy.md) defines lightweight branch names and
  when direct changes to `main` are reasonable.
- [Commits](commits.md) defines the Conventional Commits vocabulary used across
  the organization.
- [Documentation](documentation.md) assigns responsibilities to the standard
  repository documents.
- [AI-assisted engineering](ai-assisted-engineering.md) defines responsible use
  of AI tools in engineering work.

Project-specific practices belong in the relevant repository documentation. Do
not duplicate these standards unless a local exception must be explained.
